![Status](https://img.shields.io/badge/status-active-success.svg)
![AWS](https://img.shields.io/badge/AWS-Cloud%20Learning-orange.svg)

A curated, professional-grade knowledge base for mastering Amazon Web Services through structured notes, practical examples, architecture guidance, and hands-on reference material.

This repository is designed for developers, cloud engineers, architects, and certification candidates who want a reliable, organized, and production-oriented way to learn AWS from fundamentals to advanced concepts.

## Why this repository stands out

- Structured learning path from foundational cloud concepts to advanced AWS services
- Coverage across compute, storage, networking, security, databases, serverless, containers, analytics, AI, and cost optimization
- Practical notes, code snippets, and real-world project ideas that support both learning and implementation
- Built to be useful for interviews, certification preparation, and day-to-day cloud engineering work

## What you will find here

- Deep notes on core AWS services and architectural patterns
- Reference material for IAM, networking, security, monitoring, and deployment
- Hands-on examples and reusable code snippets
- Guidance for modern cloud-native design and best practices
- Curated resources for exam preparation and professional growth

## Recommended learning path

1. Start with `00-prerequisites/` and `01-getting-started/` for the foundation.
2. Move through core service areas such as IAM, networking, and storage.
3. Explore security, monitoring, observability, and deployment topics for real-world readiness.
4. Use `17-real-world-projects/` and `18-code-snippets/` to apply what you learn.
5. Leverage `19-resources-cheatsheets/` for certification and interview preparation.

## Getting started

### Prerequisites

- An AWS account (the Free Tier is sufficient for many examples)
- AWS CLI v2 installed and configured
- Basic familiarity with the command line

### Suggested workflow

- Read the notes in order if you are new to AWS
- Jump directly to a service folder when you need a reference
- Practice the examples in a safe, non-production environment
- Cross-check important details with official AWS documentation
